# 詰将棋目覚まし (tsume-alarm)

詰将棋を解かないと止められない目覚ましアプリ。Expo SDK 56 / TypeScript 製。

---

## 概要

設定した時刻に通知（ローカル通知 + サウンド）が届きます。通知をタップしてアプリを開くと詰将棋の出題が始まり、**指定問数を正解するまでアラーム音が鳴り続けます**。スヌーズなし、ヒントなし、解答表示なし——「逃げ道なし」の設計です。

---

## OS アラームの制約について（重要）

iOS / Android は、**アプリがロック中・終了状態のときに強制的に前面に出続けることを OS レベルで禁止**しています。

このアプリの実際の動作は次の通りです：

1. 設定した時刻に**ローカル通知（＋サウンド）** が届く
2. ユーザーが通知をタップすると、アプリが解答画面に遷移してアラーム音がループ再生される
3. **指定問数の詰将棋を正解する**と音が止まり、結果画面に移行する

> 通知を無視したり、アプリを強制終了したりすることは OS として可能です。  
> 「逃げ道なし」はあくまで **アプリの解答 UI 内の設計**（スヌーズボタン・解答表示・スキップが存在しない）です。

## 起動方法

```bash
# 依存インストール（初回のみ）
pnpm install

# テスト実行（全 12 スイート / 49 テスト）
pnpm test

# 開発サーバー起動
npx expo start
```

> `pnpm test` は問題データの健全性検証も兼ねています（下記「問題の追加」参照）。

---

## 設定項目

| 設定 | 説明 | デフォルト |
|------|------|-----------|
| アラーム時刻 | 通知を送る時刻 | 07:00 |
| 難易度（最小手数） | 出題する詰将棋の最小手数（奇数：1/3/5） | 1 |
| 難易度（最大手数） | 出題する詰将棋の最大手数（奇数：1/3/5） | 5 |
| 必要正解問数 | アラームを止めるのに必要な正解数 | 3 |

難易度の単位は **手数**（攻方の指し手の数）です。手数が多いほど難しくなります。

---

## 収録問題

| 手数 | 問数 | 特徴 |
|------|------|------|
| 1手詰 | 3問 | 隅玉への頭金など基本形 |
| 3手詰 | 4問 | 隅玉・中段玉への金・飛車連携 |
| 5手詰 | 5問 | 隅玉＋角/飛/馬＋合駒をからめた形 |

全 **12 問**。

---

## 問題の追加方法

1. `src/lib/puzzles.ts` の `PUZZLES` 配列にエントリを追記します。

   ```ts
   {
     id: 'm3-005',       // 一意の文字列
     plies: 3,           // 手数（奇数）
     sfen: '...',        // 攻方が手番（b）の完全 SFEN
     tags: ['頭金'],
     note: '解説（任意）',
   }
   ```

2. `pnpm test` を実行します。

   - `puzzles.test.ts` が **ちょうど N 手詰**（不詰・余詰・手数違いは NG）かつ**初手の余詰なし**を自動検証します。
   - 不健全な問題が含まれていると **テストが FAIL** し、ビルドを通せません。
   - テストが全 PASS すれば採用 OK です。

> SFEN の注意点：攻方（詰める側）が必ず手番（`b`）になるように記述してください。

## プロジェクト構成

```
tsume-alarm/
├── App.tsx              # 3画面ルーター (home / solving / result)
├── src/
│   ├── lib/             # 純粋ロジック（UI に非依存、Jest でテスト済み）
│   │   ├── moves.ts     # 合法手生成
│   │   ├── solver.ts    # AND/OR 詰み探索
│   │   ├── verify.ts    # 問題健全性検証
│   │   ├── puzzles.ts   # 収録問題一覧
│   │   ├── judge.ts     # 解答判定（玉方最善応手）
│   │   ├── session.ts   # 出題セッション管理
│   │   ├── settings.ts  # 設定の型・バリデーション・永続化
│   │   ├── alarm.ts     # ローカル通知スケジューリング
│   │   ├── stats.ts     # 成績・連続記録
│   │   ├── boardModel.ts  # 盤面状態モデル
│   │   └── interaction.ts # タップ選択・成り・打ちロジック
│   └── ui/              # React Native コンポーネント
│       ├── Board.tsx        # 盤面描画
│       ├── Hand.tsx         # 持ち駒表示
│       ├── SolveBoard.tsx   # 解答インタラクション
│       ├── HomeScreen.tsx   # ホーム・設定画面
│       ├── SolvingScreen.tsx # 出題・解答画面
│       └── ResultScreen.tsx  # 結果・連続記録画面
└── assets/              # 効果音 (alarm.wav)
```
